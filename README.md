# Wezterm Module Template

This is a demonstration of how to configure Wezterm using `nix-wrapper-modules`. 
It showcases the powerful feature of injecting Nix options directly into a standalone Lua configuration.

## File Structure

- `flake.nix`: The entry point that defines inputs (nixpkgs, nix-wrapper-modules) and outputs (the final wrapped package).
- `module.nix`: The Nix module where you define custom options (like `myConfig.color_scheme`), pass them to `luaInfo`, and specify the path to your Lua entrypoint.
- `lua/init.lua`: Your pure Lua Wezterm configuration. It pulls in the Nix values dynamically using `require('nix-info')`.

## How Nix Options are Injected

1. In `module.nix`, you map any Nix option or path to `config.luaInfo`:
   ```nix
   config.luaInfo = {
     color_scheme = config.myConfig.color_scheme;
     font_size = config.myConfig.font_size;
   };
   ```

2. Inside `lua/init.lua`, you can consume these variables using the special `nix-info` module generated by the wrapper:
   ```lua
   local nixInfo = require('nix-info')
   -- nixInfo(default_value, "path", "to", "key")
   config.color_scheme = nixInfo("AdventureTime", "color_scheme")
   ```
   This mechanism allows your `init.lua` to remain valid pure Lua that can even run outside of Nix (by falling back to the `default_value`).

## Usage

To initialize this template flake into an empty directory, run:

```bash
nix flake init -t github:BirdeeHub/nix-wrapper-modules#wezterm
```

To build and run it from this directory:

```bash
nix build .
./result/bin/wezterm
```
